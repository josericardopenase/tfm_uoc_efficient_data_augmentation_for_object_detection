## 📖 Abstract

The creation of domain-specific datasets for object detection is one of the main challenges in computer vision. Training robust detectors requires large volumes of properly annotated images with bounding boxes, which involves a costly, time-consuming process that depends on specialized manual labeling. The lack of diversity and extreme cases directly limits the performance of detection systems.

This repository contains the code and experiments for my Master's Thesis, which proposes a fully automatic methodology for generating synthetic images using **Diffusion Models (Flux)**. This approach is generalizable to any dataset and includes strategies for the automatic extraction of annotations using **Grounded SAM**, data balancing, and subsequent refinement for training lightweight edge devices (like **YOLO**).

## 🚀 Pipeline Overview

The workflow is divided into four main stages, seamlessly integrated without requiring manual annotation:

1. **Prompt Generation:** We define the dataset through configurable scene attributes (object class, environment, weather, and quantity). A language model maps these to natural language prompts.
2. **Synthetic Image Generation:** We use Latent Diffusion Models (Flux FP8 / LoRA) conditioned on the generated prompts to create a diverse and highly realistic synthetic dataset.
3. **Automatic Annotation:** **Grounded SAM** (combining Grounding DINO and Segment Anything) is applied to automatically pseudo-label the synthetic images with accurate bounding boxes.
4. **Analysis & Training:** Domain shift analysis via CLIP embeddings, followed by the training and evaluation of lightweight object detectors on the synthetic-augmented datasets.

## 📂 Repository Structure

The project is structured into modular Jupyter Notebooks representing the different stages of the pipeline:

* **`prompt_generation.ipynb`**: Logic for dataset definition and dynamic prompt generation based on scenario constraints using language models.
* **`diffusion_generation.ipynb`**: Implementation of the text-to-image synthetic generation using Flux diffusion models.
* **`groundedsam_labeling.ipynb`** (and variants): Scripts to automatically generate bounding boxes on synthetic images using Grounded SAM.
* **`change_labels.ipynb`** / **`evaluate_grounded_sam.ipynb`**: Tools for reviewing, evaluating, and refining the pseudo-labels generated by the zero-shot pipeline.
* **`dataset_balancer.ipynb`** / **`dataset_stats.ipynb`**: Scripts to analyze class distributions and synthetically balance long-tail datasets (e.g., generating underrepresented classes).
* **`dataset_transfer.ipynb`**: Utilities for formatting and transferring datasets to standard detection formats (like YOLO/COCO).
* **`domain_shift_analysis.ipynb`**: Employs CLIP embeddings and t-SNE/K-means to visualize and quantify the domain gap between real and synthetic data.
* **`charts.ipynb`**: Notebook for generating the visualizations and plots included in the thesis.
* **`training/`**, **`training_flux/`**, **`runs/detect/`**: Directories containing the training routines, configuration, and experimental results for the object detection models.

## 📊 Key Results

* **Reduced Manual Effort:** Successfully generated and pseudo-labeled datasets across multiple domains (PASCAL VOC, KITTI, African Wildlife, and QAISC Maritime) natively.
* **Data Balancing:** Transformed long-tail data distributions into balanced distributions via targeted synthetic generation, substantially improving minority class representation.
* **Performance Gain:** Demonstrated that augmenting real data with synthetic samples, combined with domain-specific LoRA fine-tuning, improves generalization capability and detector robustness.
